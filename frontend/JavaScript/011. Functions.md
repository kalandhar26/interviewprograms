# Functions

- A function is a block of code that does some work when we call it.

```javascript
function add(a, b) {
  return a + b;
}

add(2, 3); // 5
```

## Nested Function

- A nested function is a function written inside another function. Inner function can be used inside only.

```javascript
function outer() {
  function inner() {
    console.log("I am inside");
  }
  inner();
}

outer();
```

## Lexical Scoping

- Lexical scope means: A function can use variables from where it was written, not where it is called.

```javascript
function parent() {
  let toy = "Car";

  function child() {
    console.log(toy);
  }

  child();
}

parent();
// Child can see toy
```

- Parent can’t see child’s variables

## Immediately Invoked Function Expression (IIFE)

- An IIFE is a function that Runs immediately and Runs only once
- Avoid global variables
- Create private scope
- One-time setup

```javascript
(function() {
  let secret = "Hidden";
  console.log(secret);
})();
// secret is NOT accessible outside
```

## Revealing Module Pattern

- A design pattern that uses IIFE to create a private scope and returns an object with references to the private
  functions and variables that should be publicly accessible.
- Revealing Module Pattern: hide private data ,show only selected functions

- Example

```javascript
const counterModule = (function() {
  let count = 0; // private

  function increment() {
    count++;
  }

  function getCount() {
    return count;
  }

  return {
    increment: increment,
    getCount: getCount
  };
})();
```

- usage

```text
counterModule.increment();
counterModule.increment();

console.log(counterModule.getCount()); // 2
```

- Cannot access count directly.

### How IIFE Works?

- IIFE creates private space
- Lexical scope keeps variables alive
- Return object reveals public methods
